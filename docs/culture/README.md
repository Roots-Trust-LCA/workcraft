## Cultural Artifacts

### The Forest Covenant

**Founding myth:** Eight practitioners gathered in a place between watersheds and made a covenant: *Not to merge, but to stand in the forest together.*

**Use in:**
- New member orientation
- Guild convergence ceremonies
- Annual covenant renewal rituals

### Sacred Objects

Each guild has a **founding artifact**:

- **Loom:** The Loom itself — symspoietic systems architecture
- **Quill:** Codex Meridianus — text that teaches how to think
- **Lens:** Mirror of Dextera — design principle for seeing through others' eyes
- **Bell:** Bell of Harmonix — reveals what's already resonating
- **Root:** Seed Archive — living knowledge of diversity and regeneration
- **Hand:** Maker's Loom — apprenticeship space for embodied learning
- **Forge:** Phoenix Hammer — tool that demands respect and transforms with care
- **Spring:** Basin of Renewal — place where anyone can rest and be held

**Use in:**
- Guild identity work
- Ritual and ceremony design
- Visual design and branding

## Guild Relationships

### Complementary Tensions

The guilds are not in conflict, but in **dynamic relationship**:

| Guild | Strengthened By | Checked By |
|-------|-----------------|------------|
| **Loom** (Code) | Lens (Form) — make code visible | Spring (Water) — care for humans using code |
| **Quill** (Word) | Root (Earth) — ground words in reality | Forge (Fire) — accept accountability for impact |
| **Lens** (Form) | Hand (Body) — form must serve movement | Bell (Sound) — form must allow resonance |
| **Bell** (Sound) | Root (Earth) — rhythm follows nature | Hand (Body) — sound must be livable |
| **Root** (Earth) | Quill (Word) — story the land | Hand (Body) — labor with the earth |
| **Hand** (Body) | Forge (Fire) — catalyze skill | Lens (Form) — make beauty accessible |
| **Forge** (Fire) | Root (Earth) — transformation serves regeneration | Spring (Water) — accept the weight of change |
| **Spring** (Water) | All others — connection requires craft | All others — don't absorb their responsibility |

## Seasonal Rhythms

**Cultural calendar for guild activities:**

### Spring (Feb-Apr)
- **Guild of the Root awakes** — Plant, prepare
- **Guild of the Hand learns** — New skills, apprenticeships begin
- **Specialization reforging** — Change expertise paths
- **Guild chambers reconvene** — Annual planning

### Summer (May-Jul)
- **Guild of the Lens blooms** — Make visible, build
- **Guild of the Bell coordinates** — Large gatherings
- **Venture execution** — Cross-guild projects launch
- **Guild summits** — In-person gatherings

### Autumn (Aug-Oct)
- **Guild of the Forge harvests** — Transform, complete
- **Guild of the Root prepares** — Winter preparation
- **Guild of the Quill documents** — Record the season
- **Patronage calculations** — Economic harvest

### Winter (Nov-Jan)
- **Guild of the Spring activates** — Rest, reflect, heal
- **Guild of the Loom maintains** — Systems upkeep
- **Guild of the Quill writes** — Archive the year
- **Long-term learning** — Skill development

## Rituals & Ceremonies

### Recognition Ceremonies

**Contributor → Steward:**
1. Peer nominations collected
2. Contribution history reviewed
3. Domain expertise demonstrated
4. Guild ceremony (symbolic artifact presentation)
5. Chain record updated

**Steward → Principal:**
1. Leadership portfolio reviewed
2. Federation council input gathered
3. Cross-guild endorsements
4. Ceremony at guild convergence
5. Governance weight activated

### Seasonal Ceremonies

**Spring Reforging:**
- Members can change specialization
- Guild officers elected
- Annual covenants renewed
- New ventures blessed

** Autumn Harvest:**
- Venture completions celebrated
- Patronage distributions announced
- Gratitude rituals
- Winter planning

## The Enemy: Monoculture

**Cultural narrative:** In Workcraft lore, there is one real enemy — **monoculture**.

- Monoculture in agriculture depletes soil
- Monoculture in code breaks when reality changes
- Monoculture in narrative silences other voices
- Monoculture in management extracts without regenerating

**All eight guilds unite against monoculture** — in any form.

**Use in:**
- Strategic planning contexts
- Organizational change narratives
- Unity-building across guilds

## When to Use Cultural vs. Professional Language

### Cultural Language Appropriate

- ✅ Internal guild gatherings
- ✅ Onboarding for creative organizations
- ✅ Community events and rituals
- ✅ Cultural documentation
- ✅ Creative projects and expression
- ✅ Internal newsletters and updates

### Professional Language Required

- ✅ Enterprise sales materials
- ✅ HR integration documents
- ✅ Executive presentations
- ✅ External partnerships
- ✅ Compliance documentation
- ✅ Technical implementation guides

### Mixed Contexts (Use Judgment)

- ✅ Public website (professional with cultural touches)
- ✅ Community presentations (adapt to audience)
- ✅ Cross-guild coordination (professional framework, cultural warmth)
- ✅ Social media (depends on platform and audience)

## Contributing to Culture

**How to add to cultural documentation:**

1. **Propose** — Share draft in guild channel
2. **Review** — Guild stewards review for alignment
3. **Test** — Use in cultural context (ceremony, event, etc.)
4. **Adopt** — Add to `/docs/culture/` with guild attribution
5. **Practice** — Integrate into guild life

**What belongs here:**
- Origin stories and myths
- Ritual and ceremony designs
- Guild identity work
- Cultural artifacts and symbols
- Seasonal practices
- Creative expressions

**What belongs in `/docs/enterprise/`:**
- Professional frameworks
- Technical specifications
- HR integration guides
- Implementation documentation
- Compliance materials

## Relationship to Enterprise Documentation

**Two tracks, one ecosystem:**

```
┌─────────────────────────────────────────────────────┐
│                  World of Workcraft                  │
├─────────────────────────────────────────────────────┤
│  ┌──────────────────┐     ┌──────────────────┐     │
│  │   ENTERPRISE     │     │     CULTURE      │     │
│  │     TRACK        │     │      TRACK       │     │
│  ├──────────────────┤     ├──────────────────┤     │
│  │ Professional     │     │ Guild identity   │     │
│  │ HR integration   │     │ Origin stories   │     │
│  │ Technical specs  │     │ Rituals          │     │
│  │ Compliance       │     │ Symbols          │     │
│  │ Implementation   │     │ Seasonal practice│     │
```

**Both tracks are essential:**
- **Enterprise** → Enables adoption, integration, scale
- **Culture** → Builds belonging, identity, meaning

**Never let one track dominate:**
- Enterprise without culture → Hollow coordination, no belonging
- Culture without enterprise → Beautiful stories, no adoption
