## Core Vocabulary Translation

### Engagement & Work Classification

| Gaming Term | Professional Term | Definition |
|-------------|------------------|------------|
| Quest | **Engagement** or **Initiative** | A unit of work with clear scope and deliverables |
| Daily Quest | **Standing Commitment** | Routine, recurring work (operations, maintenance) |
| Side Quest | **Process Improvement** | Non-critical enhancements (refactoring, documentation) |
| Main Story Quest | **Strategic Initiative** | Core roadmap objectives, key deliverables |
| World Boss | **Critical Milestone** | Organization-wide deadline or deliverable |
| Raid | **Cross-Functional Initiative** | Multi-party coordinated work requiring role clarity |
| Party | **Working Group** or **Squad** | Small team (3-8 people) for sprint-level collaboration |
| Guild | **Practice Community** | Global professional community organized by function |
| Guild Chamber | **Practice Forum** | Standards body for a craft/function |
| Forest | **Federation** or **Network** | Ecosystem of autonomous nodes connected peer-to-peer |
| Tree | **Node** or **Hub** | Autonomous operating unit within a federation |
| Hub | **Node** | Neutral term for autonomous organizational unit |

## Engagement Type Definitions

### Standing Commitment

**Definition:** Routine, recurring work that maintains operations.

**Characteristics:**
- Predictable, recurring
- Low coordination overhead
- Essential for organizational function

**Examples:**
- System maintenance
- Customer support rotation
- Financial reporting
- Heartbeat operations

### Process Improvement

**Definition:** Non-critical enhancements that improve systems or processes.

**Characteristics:**
- Not time-critical
- Improves future capacity
- Often self-directed

**Examples:**
- Code refactoring
- Documentation updates
- Tooling improvements
- Workflow optimization

### Strategic Initiative

**Definition:** Core roadmap objectives critical to organizational success.

**Characteristics:**
- High priority
- Clear deliverables
- Timeline-bound
- Leadership visibility

**Examples:**
- Product launches
- Market expansion
- Major feature development
- Compliance deadlines

### Cross-Functional Initiative

**Definition:** Complex work requiring coordination across multiple functions.

**Characteristics:**
- Multi-party involvement
- Role clarity required
- Coordination overhead
- Shared accountability

**Examples:**
- Platform migrations
- Organizational restructuring
- Multi-team product releases
- Partnership integrations

## Coordination Structure Definitions

### Working Group

**Scope:** 3-8 people  
**Purpose:** Sprint-level collaboration  
**Duration:** Days to weeks  
**Coordination:** Daily standups, shared task board

### Practice Community

**Scope:** Global (by function)  
**Purpose:** Standards, mentorship, best practices  
**Duration:** Ongoing  
**Coordination:** Monthly forums, annual summits

### Node Bridge

**Scope:** Two nodes  
**Purpose:** Bilateral coordination  
**Duration:** As needed (can dissolve anytime)  
**Coordination:** Bridge stewards, shared channels

### Federation Council

**Scope:** Ecosystem-wide  
**Purpose:** Shared infrastructure governance  
**Duration:** Ongoing (rotated representation)  
**Coordination:** Quarterly meetings, consensus-seeking

## Economic Terms

### Contribution Credits

**Definition:** Internal currency earned through verified contributions.

**Earning Mechanisms:**
- Standing commitments (baseline)
- Process improvements (variable)
- Strategic initiatives (premium)
- Cross-functional initiatives (premium + royalty share)

**Use Cases:**
- Internal service exchange
- Resource allocation
- Patronage weight calculation

### Patronage Allocation

**Definition:** Distribution of organizational value to contributors.

**Proposed Formula:**
- 40% Labor — Verified contributions
- 30% Revenue — Value generated by ventures
- 20% Capital — Infrastructure investment
- 10% Community — Governance, mentorship

### Capacity Budget

**Definition:** Available resources for a professional per cycle.

**Components:**
- Time allocation (hours/week)
- Compute allocation (tokens, API calls)
- Attention allocation (meeting capacity)

**Management:**
- Visible before commitment
- Recovery period after intensive work
- Utilization tracking

## Usage Guidelines

### When to Use Professional Vocabulary

**Use Professional Terms:**
- Enterprise communications
- HR integration documents
- Executive presentations
- External partnerships
- Compliance documentation

**Gaming Metaphors Acceptable:**
- Internal community building
- Cultural documents
- Onboarding for creative organizations
- Community events and gatherings

### Consistency Rules

1. **Never mix vocabularies** in the same document
2. **Professional first** for external-facing materials
3. **Gaming optional** for internal culture work
4. **Always define** terms on first use in new contexts
