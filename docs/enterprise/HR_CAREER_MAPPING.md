## Recognition Levels ↔ Career Stages

### Detailed Mapping

| WoWc Standing | Contribution Threshold | HR Equivalent | Compensation Band | Governance Rights |
|---------------|----------------------|---------------|-------------------|-------------------|
| **Contributor** | 5+ verified contributions | Individual Contributor I-II | Band 1-2 | Full participation |
| **Steward** | 20+ contributions + expertise | Senior IC / Manager | Band 3-4 | Committee eligibility |
| **Principal** | 50+ contributions + leadership | Principal / Director+ | Band 5+ | Strategic input, governance weight |

### Progression Requirements

**Contributor → Steward:**
- 20+ verified contributions in professional profile
- Domain expertise demonstrated (peer review)
- 40+ days active participation
- Mentorship of 2+ contributors

**Steward → Principal:**
- 50+ verified contributions
- Demonstrated leadership (initiative ownership)
- Standards development or governance participation
- Cross-functional impact

**WoWc Capacity System:**

- Utilization tracking shows allocated vs. available

**HR Integration:**
- Capacity units map to FTE percentage

## Contribution Record ↔ Performance Documentation

### What Gets Recorded

Every contribution includes:

1. **Engagement Type** — Standing, Process Improvement, Strategic, Cross-Functional
2. **Scope & Deliverables** — Clear objectives
3. **Role** — Function performed (primary/secondary)
4. **Verification** — Peer or system attestation
5. **Capacity Invested** — Hours/units contributed
6. **Outcome** — Completed, impact measured

### Performance Review Integration

**Traditional HR:**
- Annual review
- Manager assessment
- Self-assessment
- Limited objective data

**WoWc-Enhanced:**
- Continuous contribution record
- Peer verification
- Objective contribution metrics
- Domain expertise progression visible

**Hybrid Approach:**
- Use WoWc contribution data as input to annual reviews
- Manager reviews contribution history with employee
- Domain expertise progression informs promotion discussions
- Recognition level provides objective career stage marker

## Implementation Guide

### Phase 1: Profile Mapping (Week 1-2)

1. Map existing job families to WoWc primary functions
2. Identify common secondary functions for each role
3. Assign professional profiles to all employees
4. Communicate mapping to employees

### Phase 2: Contribution Tracking (Week 3-6)

1. Configure contribution submission system
2. Train managers on verification process
3. Begin recording all work as contributions
4. Establish contribution categories by engagement type

### Phase 3: Recognition Calibration (Week 7-10)

1. Review existing compensation bands
2. Map to WoWc recognition levels
3. Calibrate contribution thresholds to current levels
4. Validate with HR leadership

### Phase 4: Full Integration (Week 11+)

1. Integrate contribution data into performance review cycle
2. Enable domain expertise selection
3. Launch practice communities
4. Begin cross-functional initiative tracking

## Compliance Considerations

### Equal Employment Opportunity

**WoWc Design:**
- Professional profiles based on skills, not protected characteristics
- Recognition levels based on verified contributions, not subjective assessment
- Domain expertise open to all who meet requirements

**HR Compliance:**
- Monitor recognition level distribution across demographics
- Ensure verification process doesn't introduce bias
- Audit domain expertise access for equity

### Compensation Equity

**WoWc Transparency:**
- All contributions visible and verifiable
- Allocation formula parameterized and transparent
- Recognition level maps to compensation bands

**HR Compliance:**
- Regular pay equity analysis by recognition level
- Contribution thresholds validated for job relevance
- Allocation formula reviewed for disparate impact

### Data Privacy

**WoWc System:**
- Contribution record is immutable
- Professional profile is portable
- Recognition level is public within ecosystem

**HR Compliance:**
- Employee consent for contribution tracking
- GDPR/CCPA compliance for data portability
- Clear data retention policies

## Case Study: Techne Studio

### Current State (Pre-WoWc)

- 8 organizers across multiple ventures
- Informal role definitions
- Annual performance reviews
- Compensation based on negotiation + market

### WoWc Integration (6 Months)

**Month 1-2:** Profile mapping
- All organizers assigned professional profiles
- Primary/secondary functions documented

**Month 3-4:** Contribution tracking
- All work recorded as contributions
- Verification process established

**Month 5-6:** Recognition calibration
- Recognition levels assigned based on contribution history
- Domain expertise paths defined

### Outcomes

- **Clarity:** Everyone sees their role and growth path
- **Objectivity:** Contribution data informs reviews
- **Retention:** Visible progression reduces turnover
- **Equity:** Transparent criteria reduce bias
