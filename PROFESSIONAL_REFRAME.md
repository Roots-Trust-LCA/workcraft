# World of Workcraft: Professional Reframe

## Core Insight

The underlying coordination patterns are sound. The gaming vocabulary creates unnecessary friction in professional contexts. This document translates WoWc into **organizational operating system** language.

---

## Revised Value Proposition

### Before (Gaming Frame)
> "World of Workcraft gamifies coordination by turning work into quests, enabling players to level up, join guilds, and earn loot through raids."

### After (Professional Frame)
> "World of Workcraft provides a coordination infrastructure that makes organizational work legible: clear engagement types, visible capability development, transparent economic flows, and structured cross-functional collaboration."

---

## Core Mechanics (Reframed)

### 1. Engagement Types

Work is categorized by scope and coordination requirements:

| Type | Description | Professional Context |
|------|-------------|---------------------|
| **Standing Commitment** | Routine, recurring work | Operations, maintenance, heartbeat tasks |
| **Process Improvement** | Non-critical enhancements | Refactoring, documentation, tooling |
| **Strategic Initiative** | Core roadmap objectives | Product development, key deliverables |
| **Cross-Functional Initiative** | Multi-party coordinated work | Complex projects requiring role coordination |

### 2. Capability Development

Professionals develop through demonstrated contribution:

- **Professional Profile** = Primary Function × Secondary Function (64 combinations)
- **Domain Expertise** unlocks after 20 contributions in a focus area
- **Contribution Credits** accumulate through verified work
- **Professional Stage** advances through contribution thresholds
- **Recognition Level** reflects community acknowledgment (Contributor → Steward → Principal)

### 3. Resource Management

- **Capacity Budget** tracks available resources (time, compute, attention)
- **Recovery Period** provides renewal after intensive work cycles
- **Resource Demand** is visible before commitment to engagements
- **Utilization Dashboard** shows current allocation vs. capacity

### 4. Economic Transparency

- **Contribution Record** is immutable and auditable
- **Allocation Framework** distributes value based on contribution type
- **Credit System** enables internal value exchange
- **Patronage Mechanism** ensures economic justice at scale

### 5. Coordination Structures

| Structure | Scope | Purpose |
|-----------|-------|---------|
| **Working Group** | 3-8 people | Sprint-level collaboration |
| **Practice Community** | Global (by function) | Standards, mentorship, best practices |
| **Node Bridge** | Two operating units | Bilateral coordination |
| **Federation Council** | Ecosystem-wide | Shared infrastructure governance |

## Why This Matters

The gaming frame creates **adoption friction** in professional contexts:
- Executives hear "game" and dismiss as trivial
- HR professionals can't map to existing career frameworks
- Legal/compliance teams struggle with "loot" and "raids"
- Enterprise sales requires explaining the metaphor before the value

The professional frame preserves **all coordination benefits** while enabling:
- Direct integration with existing organizational structures
- Clear mapping to career development frameworks
- Enterprise procurement and compliance alignment
- Serious tone without losing engagement mechanics

---

## Implementation Priority

The mechanics don't change. Only the vocabulary:

1. **Dashboard** (not HUD) — Resource visibility, engagement board
2. **Capacity Budget** (not Mana) — Prevents burnout, enables planning
3. **Engagement Types** (not Quests) — Clear scope classification
4. **Professional Profile** (not Archetype) — Career-relevant identity
5. **Contribution Record** (not Combat Log) — Auditable work history

## The Forest Covenant → Federation Principles

**Before:**
> "Each hub is a tree. Together they form a forest."

**After:**
> "Each node operates autonomously. Together they form a federation."

**Principles:**
1. **Node Autonomy** — Local governance, local parameterization
2. **Portable Identity** — Professional profile recognized across federation
3. **Shared Primitives** — Common infrastructure (credits, chain, patronage)
4. **Async-First Coordination** — Local-first, global when needed
5. **Data Sovereignty** — Each node holds its own data
